# probe-aeneas

Cross-language merge tool for [Aeneas](https://github.com/AeneasVerif/aeneas)-transpiled
projects. Bridges the gap between `probe-rust` (Rust atoms) and `probe-lean` (Lean atoms)
by generating translation mappings and producing a combined call graph.

## Overview

Given a Rust project atomized by `probe-rust` and its Aeneas-transpiled Lean counterpart
extracted by `probe-lean`, `probe-aeneas` generates a merged atom graph where Rust functions
have dependency edges crossing into their Lean counterparts.

## Usage

### Full pipeline (from project paths)

The simplest way: point `probe-aeneas` at the Rust and Lean project directories.
It will run `probe-rust extract` and `probe-lean extract` automatically (installing
them if needed), generate `functions.json` via `lake exe listfuns`, and merge everything.

```bash
probe-aeneas merge \
  --rust-project path/to/rust/project \
  --lean-project path/to/lean/project \
  --output merged.json
```

### Full pipeline (from pre-generated JSONs)

If you've already run the extractors, pass the JSON files directly. You must provide
`--functions` since there's no Lean project to auto-generate it from.

```bash
probe-aeneas merge \
  --rust path/to/rust_atoms.json \
  --lean path/to/lean_atoms.json \
  --functions path/to/functions.json \
  --output merged.json
```

### Mixed mode

You can mix project paths and JSON files:

```bash
probe-aeneas merge \
  --rust-project path/to/rust/project \
  --lean path/to/lean_atoms.json \
  --functions path/to/functions.json \
  --output merged.json
```

### Generate translations only

```bash
probe-aeneas translate \
  --rust path/to/rust_atoms.json \
  --lean path/to/lean_atoms.json \
  --functions path/to/functions.json \
  --output translations.json
```

### Generate functions.json only

```bash
probe-aeneas listfuns \
  --lean-project path/to/lean/project \
  --output functions.json
```

## Translation strategies

Translations are generated by matching Rust atoms to Lean atoms via `functions.json`
(which maps Rust qualified names to Lean names). Three strategies are used in priority order:

1. **rust-qualified-name**: Exact match via the `rust-qualified-name` extension field
   on Rust atoms (derived from Charon LLBC), joined with `functions.json` `rust_name`.
2. **file + display-name**: Same source file path + matching base method name.
3. **file + line overlap**: Same source file + overlapping line ranges.

## Auto-install

When `--rust-project` or `--lean-project` is used, `probe-aeneas` will automatically
find or install the required tools:

- **probe-rust**: checked on PATH, then `~/.cargo/bin/`. If not found, installed via
  `cargo install --git https://github.com/Beneficial-AI-Foundation/probe-rust.git`.
- **probe-lean**: checked on PATH, then `~/.local/bin/`. If not found, cloned and built
  with `lake build`, then copied to `~/.local/bin/probe-lean`.

## Requirements

- **Rust toolchain** (`cargo`) for building/installing `probe-rust`
- **Lean 4 toolchain** (`elan`, `lake`) for building/installing `probe-lean` and running `listfuns`
- `probe-rust` and `probe-lean` are auto-installed when using `--rust-project` / `--lean-project`
