# probe-aeneas

Cross-language merge tool for [Aeneas](https://github.com/AeneasVerif/aeneas)-transpiled
projects. Bridges the gap between `probe-rust` (Rust atoms) and `probe-lean` (Lean atoms)
by generating translation mappings and producing a combined call graph.

## Overview

Given a Rust project atomized by `probe-rust` and its Aeneas-transpiled Lean counterpart
extracted by `probe-lean`, `probe-aeneas` generates a merged atom graph where Rust functions
have dependency edges crossing into their Lean counterparts.

## Usage

### Full pipeline

```bash
probe-aeneas merge \
  --rust path/to/rust_atoms.json \
  --lean path/to/lean_atoms.json \
  --lean-project path/to/lean/project \
  --output merged.json
```

This runs `lake exe listfuns` to generate `functions.json`, builds a translations
file, and merges the atoms.

### Generate translations only

```bash
probe-aeneas translate \
  --rust path/to/rust_atoms.json \
  --lean path/to/lean_atoms.json \
  --functions path/to/functions.json \
  --output translations.json
```

### Generate functions.json only

```bash
probe-aeneas listfuns \
  --lean-project path/to/lean/project \
  --output functions.json
```

## Translation strategies

Translations are generated by matching Rust atoms to Lean atoms via `functions.json`
(which maps Rust qualified names to Lean names). Three strategies are used in priority order:

1. **rust-qualified-name**: Exact match via the `rust-qualified-name` extension field
   on Rust atoms, joined with `functions.json` `rust_name`.
2. **file + display-name**: Same source file path + matching base method name.
3. **file + line overlap**: Same source file + overlapping line ranges.

## Requirements

- `lake` on PATH (for `listfuns` and `merge` subcommands)
- `probe-rust` atoms JSON (input)
- `probe-lean` atoms JSON (input)
